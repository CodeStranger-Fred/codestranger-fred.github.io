Integrating Hibernate Validation in Spring Boot for Advanced Parameter Validation — Method-level Validation

While field validation is great for object properties, sometimes we also need to validate method parameters, such as query parameters or request body values. Spring provides an easy way to validate method parameters using the @Validated annotation and a special interceptor.

Steps to Enable Method Parameter Validation:

  1. Enable the Method Validation Interceptor:
@Bean
public MethodValidationPostProcessor methodValidationPostProcessor() {
    return new MethodValidationPostProcessor();
}

2. Annotate the Controller with @Validated:

import javax.validation.constraints.NotBlank;
import javax.validation.constraints.NotNull;
import javax.validation.constraints.Size;
import org.hibernate.validator.constraints.Range;

public interface GroupA {
}

public interface GroupB {
}

public class Person {

    @NotBlank(groups = {GroupA.class}, message = "User ID must not be blank")
    @Range(min = 1, max = Integer.MAX_VALUE, groups = {GroupA.class}, message = "User ID must be greater than 0")
    private Integer userId;  // User ID

    @NotBlank(groups = {GroupB.class}, message = "Username must not be blank")
    @Size(min = 4, max = 20, groups = {GroupB.class}, message = "Username must be between 4 and 20 characters")
    private String userName;  // Username

    @NotBlank(groups = {Default.class}, message = "Age must not be blank")
    @Range(min = 0, max = 100, groups = {Default.class}, message = "Age must be between 0 and 100")
    private Integer age;  // Age

    @NotNull(groups = {GroupB.class}, message = "Address cannot be null")
    private String address;  // Address

    // Getters and Setters
    public Integer getUserId() {
        return userId;
    }

    public void setUserId(Integer userId) {
        this.userId = userId;
    }

    public String getUserName() {
        return userName;
    }

    public void setUserName(String userName) {
        this.userName = userName;
    }

    public Integer getAge() {
        return age;
    }

    public void setAge(Integer age) {
        this.age = age;
    }

    public String getAddress() {
        return address;
    }

    public void setAddress(String address) {
        this.address = address;
    }
}

In the above Person class, three validation groups are used:

  • GroupA: Validates the userId field.
  • GroupB: Validates the userName and address fields.
  • Default: Validates the age field, which is the default validation group.

Scenario: Validate Specific Groups (e.g., GroupA)

For a situation where you only want to validate certain fields based on the group, you can use the @Validated annotation with the specific group.

Controller for Adding a User:

import org.springframework.validation.BindingResult;
import org.springframework.validation.annotation.Validated;
import org.springframework.web.bind.annotation.*;

@RestController
public class UserController {

    // Create user - validate only GroupA
    @PostMapping("/createuser2")
    public String createUser(@Validated(GroupA.class) @RequestBody Person person, BindingResult bindingResult) {
        if (bindingResult.hasErrors()) {
            return bindingResult.getFieldError().getDefaultMessage();  // Return the error message
        }
        // Proceed with creating the user if no validation errors
        return "User created successfully";
    }
}
