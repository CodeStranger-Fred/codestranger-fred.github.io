Integrating Hibernate Validation in Spring Boot for Advanced Parameter Validation — Global Exception Handling

Global Exception Handling with @ControllerAdvice

Handling validation errors centrally makes the code cleaner and more maintainable. You can use @ControllerAdvice and @ExceptionHandler to globally handle validation errors across all controllers.

import org.springframework.http.HttpStatus;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.*;
import org.springframework.validation.ObjectError;
import org.springframework.web.bind.annotation.ExceptionHandler;
import org.springframework.web.bind.annotation.ControllerAdvice;
import org.springframework.validation.BindingResult;
import org.springframework.beans.factory.annotation.Autowired;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import java.util.HashMap;
import java.util.Map;
import org.springframework.util.AnnotationUtils;
@ControllerAdvice
public class GlobalExceptionHandler {
    private static final Logger LOGGER = LoggerFactory.getLogger(GlobalExceptionHandler.class);
    /**
     * Handles all unknown exceptions.
     *
     * @param e the exception
     * @return a map containing the error message
     */
    @ExceptionHandler(Exception.class)
    @ResponseBody
    public Map<String, Object> handleException(Exception e) throws Exception {
        LOGGER.error(e.getMessage(), e);
        // If the exception is annotated with @ResponseStatus, rethrow the exception to handle it with the status code.
        if (AnnotationUtils.findAnnotation(e.getClass(), ResponseStatus.class) != null) {
            throw e;
        }
        Map<String, Object> result = new HashMap<>();
        result.put("Error Message: ", e.getMessage());
        return result;
    }
    /**
     * Handles business-specific exceptions.
     *
     * @param e the business exception
     * @return a map containing the error message
     */
    @ExceptionHandler(UserNotExistException.class)
    @ResponseBody
    public Map<String, Object> handleBusinessException(UserNotExistException e) {
        LOGGER.error(e.getMessage(), e);
        Map<String, Object> result = new HashMap<>();
        result.put("Error Message: ", e.getMessage());
        return result;
    }
    /**
     * Handles validation errors from method arguments.
     *
     * @param e the validation exception
     * @return a map containing the validation error messages
     */
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseBody
    public Map<String, Object> handleMethodArgumentNotValidException(MethodArgumentNotValidException e) {
        LOGGER.error(e.getMessage(), e);
        Map<String, Object> result = new HashMap<>();
        String messages = "";
        // Loop through all validation errors and concatenate their messages.
        for (ObjectError objectError : e.getBindingResult().getAllErrors()) {
            messages += objectError.getDefaultMessage() + ",";
        }
        result.put("Error Message: ", messages);
        return result;
    }
}

This will capture any validation exceptions across your entire application, making it easier to manage error responses.

Common Validation Annotations

Here are some commonly used Hibernate Validator annotations:

  • @NotBlank: Ensures the field is not null or empty.
  • @NotNull: Ensures the field is not null.
  • @Min: Specifies the minimum value.
  • @Max: Specifies the maximum value.
  • @Size: Validates the size of a string or array.
  • @Email: Validates if the field is a valid email.
  • @Length: Checks the length of a string (used for more complex cases).
