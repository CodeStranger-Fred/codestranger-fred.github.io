Integrating Hibernate Validation in Spring Boot for Advanced Parameter Validation — Handling Error Messages

Configuring Error Messages in Spring Boot

In a Spring Boot application, you can customize the error messages that are returned for validation annotations by creating a ValidationMessages.properties file. This file should be placed in the resources directory of your project, and it will allow you to externalize the error messages for validation annotations, making them easier to manage and translate.

Steps:

  1. Create the ValidationMessages.properties File:
  • In the src/main/resources directory of your Spring Boot project, create a file named ValidationMessages.properties.
  • The file should contain key-value pairs for each validation message you want to customize. The keys correspond to the placeholders used in the @NotBlank or other validation annotations.

2. Example of ValidationMessages.properties:

# ValidationMessages.properties
# Custom error message for @NotBlank validation on the 'name' field
user.name.notBlank = The name field cannot be blank.
@NotBlank(message = "{user.name.notBlank}")
private String username;

This ValidationMessages.properties file should be placed in the resources folder, as Spring Boot automatically reads this file from the classpath.

3. Using Custom Error Messages in Java Code:

In your Java code, you can use the keys defined in ValidationMessages.properties for validation annotations like @NotNull, and others.

import javax.validation.constraints.NotBlank;

public class User {

    @NotBlank(message = "{user.name.notBlank}")
    private String name;

    // Getter and Setter methods

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }
}

In this example, the @NotBlank annotation on the name field uses a message placeholder ({user.name.notBlank}). Spring Boot will automatically look for this key in the ValidationMessages.properties file and use the associated message (The name field cannot be blank.) when validation fails.

Important Notes:

  • File Naming: The file must be named exactly ValidationMessages.properties. Spring Boot looks for this file by default to load the validation messages.
  • Message Resolution: Spring Boot uses the default message source configuration to resolve the keys from the ValidationMessages.properties file. This means that the messages are externalized, and you can easily change them without modifying the code.

Example of Handling Validation Errors in a Controller:

When you perform validation in a Spring Boot controller, if the validation fails, the custom messages from ValidationMessages.properties will be automatically applied.

import org.springframework.validation.BindingResult;
import org.springframework.web.bind.annotation.*;
import javax.validation.Valid;

@RestController
public class UserController {

    @PostMapping("/createUser")
    public String createUser(@Valid @RequestBody User user, BindingResult bindingResult) {
        if (bindingResult.hasErrors()) {
            // Return the first validation error message
            return bindingResult.getFieldError().getDefaultMessage();
        }
        // Proceed with saving the user if no validation errors
        return "User created successfully";
    }
}

In the above controller method:

  • The @Valid annotation triggers validation for the User object.
  • If any validation errors occur, Spring Boot will automatically use the messages defined in ValidationMessages.properties.
